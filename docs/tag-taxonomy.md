## Tag pillar mechanics

Each tag gets a page at `/topic/[tag]` listing:
- All case studies tagged with it (chronological, newest first)
- All articles tagged with it (chronological, newest first)
- All labs projects tagged with it (chronological, newest first)

Once a tag pillar has 5+ items, it gets a 200-word intro paragraph ‚Äî that intro becomes the "pillar" content that ranks in search and AEO.

Tags are multi-select on each entry ‚Äî a case study can have 3‚Äì5 tags, an article 2‚Äì4, a lab project 2‚Äì3.

## Notes

- No categories (domain taxonomy) on `/writing` or `/labs` ‚Äî categories are exclusive and sector-bound, only used for work case studies. Writing and labs use tags only.
- Case studies use both categories (exclusive) and tags (multi). A piece tagged `regulatory-service-design` also gets a category like `regulated-industry` or `government`.
- Tags are curated at migration time ‚Äî not invented on the fly. Stick to this pool.
- If a new tag emerges (e.g., a new method area), add it to the backlog for v1.5+ discussion, don't create ad-hoc.
